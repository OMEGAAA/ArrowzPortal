ARROWZポータルサイト 運用マニュアル

このマニュアルでは、ARROWZウェブサイトの主要コンテンツ（お知らせ、ランキング、今週のメニュー、スケジュール）の更新手順について説明します。
すべての更新作業は専用の「管理画面ページ（adminページ）」を使用して行います。

==================================================
【共通の基本フロー】
1. 対応する管理画面のHTMLファイルをブラウザで開く
2. 画面上で内容を入力・編集・確認する
3. 「DOWNLOAD」ボタンを押して、設定ファイル（.js）を保存する
4. ダウンロードしたファイルをプロジェクトの `js` フォルダ内に【上書き保存】する
5. 変更内容を GitHub にプッシュ（公開）する
==================================================

1. お知らせ (NEWS) の更新
--------------------------------------------------
■ 使用する管理画面：
`news_admin.html` をブラウザで開きます。

■ 更新手順：
1. 画面左側のフォームに「DATE (日付)」「CATEGORY (カテゴリ)」「TITLE (タイトル)」「EXCERPT (抜粋)」「IMAGE STYLE (画像のスタイル)」を入力・選択します。
2. 「POST ARTICLE」ボタンを押します。（右側の現在のリストに追加されます）
   ※不要な記事がある場合は、リストの「DELETE」ボタンで消去できます。
3. 画面右上の「DOWNLOAD news_data.js」ボタンを押し、ファイルをダウンロードします。
4. ダウンロードした `news_data.js` を、プロジェクト内の `js/news_data.js` に上書き保存します。
5. GitHubに変更をプッシュして公開します。


2. ランキング (RANKING) の更新
--------------------------------------------------
■ 使用する管理画面：
`ranking_admin.html` をブラウザで開きます。

■ 更新手順：
1. 現在のデータを元にする場合は、「DOWNLOAD CSV」ボタンを押して現在のランキングデータをCSV形式でダウンロードし、Excel等で編集して保存します。
   （※あらかじめ用意されたデータ更新用CSVがある場合はそれを使用します）
2. 管理画面の「2. IMPORT CSV & UPDATE」の「ファイルを選択」から、更新用のCSVファイルを選択します。
3. 「PROCESS CSV & GENERATE JS」ボタンを押します。
4. 処理が成功すると件数が表示されるので、その下に出現する「DOWNLOAD ranking_data.js」ボタンを押してファイルをダウンロードします。
5. ダウンロードした `ranking_data.js` を、プロジェクト内の `js/ranking_data.js` に上書き保存します。
6. GitHubに変更をプッシュして公開します。


3. 今週のメニュー・月間テーマ (WEEKLY) の更新
--------------------------------------------------
■ 使用する管理画面：
`weekly_admin.html` をブラウザで開きます。

■ 更新手順：
1. 以下の各項目をフォームに入力・編集します。
   - 1. THIS WEEK'S MENU (今週のメニューの説明文と、ボタンのテキスト)
   - 2. MONTHLY THEME (月間テーマの説明文)
   - 3. CLOSED DAYS (休館日に関するお知らせなど)
2. 入力が完了したら、画面一番下の「DOWNLOAD weekly_data.js」ボタンを押してファイルをダウンロードします。
3. ダウンロードした `weekly_data.js` を、プロジェクト内の `js/weekly_data.js` に上書き保存します。
4. GitHubに変更をプッシュして公開します。


4. スケジュール (SCHEDULE) の更新
--------------------------------------------------
■ 使用する管理画面：
`schedule_admin.html` をブラウザで開きます。

■ 更新手順：
1. フォームに「DATE (YYYY.MM.DD形式)」「TYPE (休館日, イベント, お知らせ等の種類)」「TITLE (タイトル)」「Details (詳細)」を入力します。
2. 「ADD SCHEDULE」ボタンを押してリストに追加します。
   ※既存のスケジュールの「EDIT(編集)」「DELETE(削除)」もリストから操作可能です。
3. 「DOWNLOAD JS」ボタンを押し、ファイルをダウンロードします。
4. ダウンロードした `schedule_data.js` を、プロジェクト内の `js/schedule_data.js` に上書き保存します。
5. GitHubに変更をプッシュして公開します。


【よくある質問・トラブルシューティング】
Q. ファイルを上書きしてGitHubにプッシュしたのに表示が変わりません。
A. ブラウザのキャッシュが原因の可能性があります。「スーパーリロード (Windows: Ctrl + F5, Mac: Cmd + Shift + R)」をお試しください。また、jsフォルダ内のファイルが正しい場所に正しく「上書き」されているかを再度ご確認ください。
