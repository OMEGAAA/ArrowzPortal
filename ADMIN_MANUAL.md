# サイト更新マニュアル (Admin Manual)

このマニュアルでは、ARROWZウェブサイトの主要な更新作業「お知らせ」「ランキング」「今週のメニュー」「スケジュール」の更新方法について説明します。

---

## 【共通の基本フロー】
1. 対応する管理画面のHTMLファイルをブラウザで開く
2. 画面上で内容を入力・編集・確認する
3. 「DOWNLOAD」ボタンを押して、設定ファイル（.js）を保存する
4. ダウンロードしたファイルをプロジェクトの `js` フォルダ内に**【上書き保存】**する
5. 変更内容を GitHub にプッシュ（公開）する

---

## 1. お知らせの更新 (News)
管理画面 (`news_admin.html`) を使用して更新します。

### 手順
1. **記事を作成・編集する**: 画面左側のフォームに「DATE (日付)」「CATEGORY (カテゴリ)」「TITLE (タイトル)」「EXCERPT (抜粋)」「IMAGE STYLE (画像のスタイル)」を入力・選択します。
2. **リストに追加**: 「POST ARTICLE」ボタンを押します。（右側の現在のリストに追加されます）※不要な記事がある場合は、リストの「DELETE」ボタンで消去できます。
3. **ファイルのダウンロード**: 画面右上の「DOWNLOAD news_data.js」ボタンを押し、ファイルをダウンロードします。
4. **ファイルを更新**: ダウンロードした `news_data.js` を、プロジェクト内の `js/news_data.js` に上書き保存します。
5. **公開**: GitHubに変更をプッシュします。

---

## 2. ランキングの更新 (Ranking)
管理画面 (`admin.html` 内の「ランキング管理」タブ) を使用して更新します。

### 手順
1. **測定データ（Excel/CSV）の用意**: Excelファイル（`フィールドテストデータ ランキング.xlsm` など）または CSV ファイルを用意します。
2. **ファイルのインポート**: 管理画面の「2. IMPORT NEW DATA & PROCESS」のファイル選択部分から、用意したファイルを選択します。
3. **Excel解析オプションの設定（Excelの場合のみ）**: 
   Excel ファイル（.xlsx, .xlsm）を選択すると、自動的に解析オプションが表示されます。
   - **シート名**: 測定データが入力されているシート名（デフォルトは「データ」）を入力します。
   - **データ開始行（スキップする行数）**: Excelファイルの構造に応じて、データ開始前のスキップ行数を選択します。
     - **0行**（1行目がヘッダー、2行目からデータ）: **[推奨]** ほとんどの標準的なファイルでこれを使用します（例: 青木里依奈さんなど1行目から全データを含める場合）。
     - **3行**（4行目からデータ）: 過去のPythonスクリプトによる抽出と同一のデータにする場合や、最初の数行に不要なヘッダーや空行がある場合に使用します。
4. **データ処理**: 「ファイルを処理してJSを生成」ボタンを押します。
5. **ファイルのダウンロード**: 処理が成功すると件数が表示されるので、その下に出現する「DOWNLOAD ranking_data.js」ボタンを押してファイルをダウンロードします。
6. **ファイルを更新**: ダウンロードした `ranking_data.js` を、プロジェクト内の `js/ranking_data.js` に上書き保存します。
7. **公開**: GitHubに変更をプッシュします。

## 3. 今週のメニュー・月間テーマの更新 (Weekly)
管理画面 (`weekly_admin.html`) を使用して更新します。

### 手順
1. **入力・編集**: 以下の各項目をフォームに入力・編集します。
   - **THIS WEEK'S MENU**: 今週のメニューの説明文と、ボタンのテキスト
   - **MONTHLY THEME**: 月間テーマの説明文
   - **CLOSED DAYS**: 休館日に関するお知らせなど
2. **ファイルのダウンロード**: 入力が完了したら、画面一番下の「DOWNLOAD weekly_data.js」ボタンを押してファイルをダウンロードします。
3. **ファイルを更新**: ダウンロードした `weekly_data.js` を、プロジェクト内の `js/weekly_data.js` に上書き保存します。
4. **公開**: GitHubに変更をプッシュします。

---

## 4. スケジュール (休館日・イベント) の更新
カレンダーとスケジュール一覧の更新は、専用の管理画面 (`schedule_admin.html`) を使用します。

### 手順
1. **入力・編集**: フォームに「DATE (YYYY.MM.DD形式)」「TYPE (休館日, イベント, お知らせ等の種類)」「TITLE (タイトル)」「Details (詳細)」を入力します。
2. **リストに追加**: 「ADD SCHEDULE」ボタンを押してリストに追加します。※既存のスケジュールの「EDIT(編集)」「DELETE(削除)」もリストから操作可能です。
3. **ファイルのダウンロード**: 「DOWNLOAD JS」ボタンを押し、ファイルをダウンロードします。
4. **ファイルを更新**: ダウンロードした `schedule_data.js` を、プロジェクト内の `js/schedule_data.js` に上書き保存します。
5. **公開**: GitHubに変更をプッシュします。

## トラブルシューティング

### Q. ファイルを上書きしてGitHubにプッシュしたのに表示が変わらない
A. ブラウザのキャッシュが原因の可能性があります。「スーパーリロード (Windows: Ctrl + F5, Mac: Cmd + Shift + R)」をお試しください。また、jsフォルダ内のファイルが正しい場所に正しく「上書き」されているかを再度ご確認ください。

### Q. 画像が表示されない
A. アップロードした画像が表示されない場合、データが破損している可能性があります。もう一度画像をアップロードし直して、ファイルを書き出してください。
